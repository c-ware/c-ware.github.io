A collection of libraries, data structures, and other useful programs. The project
has an emphasis on offline documentation, memory efficiency, portability, and
independence. Libraries and other tools that are made to be built onto each other
are designed to be standalone, and not depend on each other for functionality.

Internally, to reduce boiler plate, many projects have components that are created
on the fly by external shell scripts. This allows for things like Makefiles, shell
scripts, and common directories to be kept consistent across all projects. This also
makes it much easier to for projects to be consistent with each other.
