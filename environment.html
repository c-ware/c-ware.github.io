Something that should be considered on occasion when writing any sort of software
or library is that of naming. Naming is important, and can make your code easier to
use or extend. However, it is also important when it comes to documentation and interfaces
that programmers use to interact with your library or software.

Because of this, in the case where someone using our code is worried about name collisions,
we provide a way to help make name collisions with things like manuals, and headers a little
less worrisome.

This 'environment' can be generated by running the command below in your terminal: curl https://c-ware.github.io/scripts/make-environment.sh | sudo bash What the script does is as follows:
  • Create a new manual category, and modifies mandb to add a new section.
  • Create a folder to hold project headers.
